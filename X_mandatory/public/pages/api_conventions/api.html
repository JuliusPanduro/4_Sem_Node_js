Singleton and Collection Resources

A resource can be a singleton or a collection.

For example:
“customers” is a collection resource and “customer” is a singleton resource (in a movie theater domain).

We can identify “customers” collection resource using the URI “/customers“. We can identify a single “movie” resource using the URI “/customers/{customerId}“.

Collection and Sub-collection Resources

A resource may contain sub-collection resources also.

For example:
sub-collection resource “tickets” of a particular “customer” can be identified using the URN “/customers/{customerId}/tickets” (in a movie theater domain).

Similarly, a singleton resource “ticket” inside the sub-collection resource “tickets” can be identified as follows: “/customers/{customerId}/tickets/{ticketId}“.

Best Practices is to use nouns to represent resources
